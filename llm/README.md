# Утилиты для работы с LLM (Large Language Models)

Директория содержит инструменты для подготовки данных к использованию с большими языковыми моделями.

## Содержимое

### `dir2prompt.py`

**Назначение:** Конвертация директории с файлами в JSON-формат и обратно для удобной передачи в контекст LLM.

**Описание:**
Скрипт позволяет "упаковать" содержимое директории (только текстовые файлы, кодировка UTF-8) в единый JSON-файл, который можно отправить в запросе к LLM. Также поддерживается обратная операция — восстановление файлов из JSON.

**Возможности:**
- Автоматическое определение текстовых файлов (бинарные помечаются как `binaryfile`)
- Игнорирование скрытых файлов/папок (можно включить флагом)
- При восстановлении — проверка на перезапись с запросом подтверждения

**Режимы работы:**

**archive** — создание JSON из директории:
```bash
./dir2prompt.py archive /path/to/dir output.json
./dir2prompt.py archive /path/to/dir output.json --hidden  # включить скрытые файлы
```

**extract** — восстановление файлов из JSON:
```bash
./dir2prompt.py extract output.json /path/to/restore
./dir2prompt.py extract output.json /path/to/restore --yes   # авто-перезапись
./dir2prompt.py extract output.json /path/to/restore --hidden  # восстанавливать скрытые
```

**Пример JSON на выходе:**
```json
{
  "script.sh": {
    "content": "#!/bin/bash\necho 'Hello'"
  },
  "image.png": {
    "content": "binaryfile"
  }
}
```

### `convert-tg-chats.py`

**Назначение:** Конвертация экспортированного JSON-файла чата Telegram в структурированный текстовый формат с группировкой по дням и информацией об участниках.

**Описание:**
Скрипт обрабатывает JSON, полученный через экспорт данных Telegram (например, `result.json`), и создаёт текстовый файл, где каждое сообщение представлено в виде:

```
[ЧЧ:ММ] U01: Текст сообщения [🔥×2] ↩U02:«Цитируемый текст»
```

**Основные возможности:**
- Нумерация активных участников (U01, U02, ... по убыванию количества сообщений)
- Разделители по дням (`── 31.12.2023 ──`)
- Фильтрация по дате
- Включение/исключение реакций и ответов (реплаев)
- Дедупликация сообщений по ID
- Игнорирование сообщений без текста

**Использование:**
```bash
./convert-tg-chats.py result.json -o chat_summary.txt
```

**Параметры:**
- `input` — путь к JSON-файлу (обязательный)
- `-o, --output` — выходной файл (если не указан, вывод в stdout)
- `--from YYYY-MM-DD` — начальная дата
- `--to YYYY-MM-DD` — конечная дата
- `--no-reactions` — не включать реакции
- `--no-replies` — не включать цитаты ответов

**Примеры:**
```bash
# Весь чат с реакциями и ответами
./convert-tg-chats.py chat.json -o chat.txt

# Только за январь 2024, без реакций
./convert-tg-chats.py chat.json -o january.txt --from 2024-01-01 --to 2024-01-31 --no-reactions

# Вывод в консоль, только сообщения с текстом
./convert-tg-chats.py chat.json --no-reactions --no-replies
```

**Формат вывода:**
```
# Название чата

## Участники
U01 — Иван Иванов
U02 — Петр Петров
U03 — Сидор Сидоров

## Сообщения

── 01.01.2024 ──
[10:15] U01: Всем привет! [👍×3]
[10:16] U02: Привет! ↩U01:«Всем привет!»
[10:20] U03: И вам не болеть

── 02.01.2024 ──
[09:00] U01: Новый день

# Итого: 4 сообщения
```

**Зависимости:**
- Python 3.6+ (только стандартная библиотека, дополнительные пакеты не требуются)

### `utoch-nya-tor.py`

**Назначение:** Самопроверка и итеративное улучшение ответов LLM через llama.cpp API.

**Описание:**
Скрипт запускает HTTP-сервер, который принимает запросы, отправляет их в llama.cpp, проверяет качество ответа и при необходимости улучшает его через несколько итераций.

**Алгоритм работы:**
1. Получение исходного ответа от LLM
2. Оценка ответа по шкале 0-10 через отдельный промпт
3. Если оценка ниже порога (по умолчанию 7) — генерация улучшенного ответа с учётом критики
4. Повтор до 3 попыток или достижения порога

**Особенности:**
- Работает как прокси-сервер, совместимый с API llama.cpp
- Цветной вывод в консоль для мониторинга
- Настраиваемое количество попыток и порог качества
- Детальный лог (уровни -v, -vv, -vvv)

**Использование:**
```bash
# Запуск сервера
./utoch-nya-tor.py --llama-host localhost --llama-port 11434 --server-port 8001

# С подробным логированием
./utoch-nya-tor.py -vvv

# С настройкой количества попыток
./utoch-nya-tor.py --max-retries 5 --retry-delay 2
```

**Пример запроса к серверу:**
```bash
curl http://localhost:8001/completion \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Объясни теорию относительности"}'
```

**Параметры:**
- `--llama-host` — хост llama.cpp API (по умолчанию localhost)
- `--llama-port` — порт llama.cpp API (по умолчанию 11434)
- `--server-port` — порт сервера (по умолчанию 8001)
- `--max-retries` — попытки при ошибках сервиса (по умолчанию 10)
- `--retry-delay` — задержка между попытками (по умолчанию 1с)
- `-v, -vv, -vvv` — уровень подробности
