# Вспомогательные файлы (support_files)

Директория содержит текстовые файлы со списками и конфигурациями, используемыми другими скриптами.

## Содержимое

### `git_repos`

**Назначение:** Список организаций и репозиториев для массового клонирования/зеркалирования.

**Описание:**
Текстовый файл, в котором каждая строка содержит имя организации или репозитория в формате `owner/repo`. Используется скриптами типа `git_org_mirroring.sh` для определения источников.

**Формат:**
- Простые имена организаций (например, `python`, `nginx`)
- Репозитории с указанием владельца (например, `pikvm/pikvm`)
- Возможны комментарии? (судя по наличию `#` в некоторых строках, да)

**Пример содержимого:**
```
0x90
ansible
pikvm/pikvm
w23
```

---

### `yt_links`

**Назначение:** Список ссылок на YouTube-плейлисты и видео для скачивания.

**Описание:**
Файл содержит структурированный список ссылок, сгруппированных по категориям:
- Документальное (Намедни, Редакция, Минаев)
- Образовательное (лекции МИФИ, алгоритмы, химия)
- Развлекательное (TheBadComedian, Масаня)
- Музыка (клипы, сеты, OST)

**Формат:**
Строки, начинающиеся с `#`, считаются комментариями/заголовками. Основные ссылки — каналы, плейлисты или отдельные видео.

**Использование:**
Файл подготовлен для использования с `youtube-dl` или `yt-dlp`. Пример команды закомментирован в начале файла:
```bash
youtube-dl --no-call-home --continue --playlist-reverse --ignore-errors --no-check-certificate --cookies cook --no-playlist -f bestvideo+bestaudio --format mp4 --batch-file yt_links --output "/mnt/data/www/video/youtube/%(channel)s/%(title)s.%(ext)s"
```
