# Генераторы плейлистов для nginx (nginx_autoindex)

Директория содержит скрипты для автоматического создания плейлистов видео и аудиофайлов в структуре директорий, предназначенных для раздачи через nginx.

## Содержимое

### `videojsgen`

**Назначение:** Генерация плейлистов для Video.js с автоматическим созданием превью.

**Описание:**
Скрипт рекурсивно обходит директории, находит видеофайлы (mp4, mov, m4v) и создаёт в каждой директории файл `.playlist.js`, совместимый с [videojs-playlist-ui](https://github.com/videojs/videojs-playlist-ui).

**Дополнительно:**
- Для каждого видео генерируется превью (один кадр в `.webp` и анимированное превью в `.webm` из нескольких кадров)
- Превью сохраняются в поддиректорию `.meta`
- Можно настроить через файл `.playlist_config.js` в каждой директории

**Конфигурация `.playlist_config.js`:**
```javascript
{
  "include_dirs": true,   // включать видео из поддиректорий
  "random": true          // случайный порядок в плейлисте
}
```

**Переменные окружения (нужно задать перед запуском):**
- `www_root` — корневая директория веб-сервера (для вырезания из путей)
- `prefix_url` — URL-префикс для подстановки в ссылки

**Пример использования:**
```bash
export www_root="/var/www"
export prefix_url="/videos"
./videojsgen
```

### `webampgen`

**Назначение:** Генерация/обновление плейлистов для Webamp (Winamp в браузере).

**Описание:**
Скрипт создаёт или обновляет файл `.webamp.js` в каждой директории с аудиофайлами (mp3, wav, flac, m4a, webm).

**Особенности:**
- Инкрементальное обновление — добавляются только новые треки
- Извлекает метаданные: длительность, исполнитель, название (через `ffprobe`)
- Генерирует JavaScript-код для инициализации Webamp

**Зависимости:** `ffprobe` (из пакета ffmpeg)

**Пример использования:**
```bash
./webampgen
```

**Структура генерируемого файла:**
```javascript
const Webamp = window.Webamp;
const webamp = new Webamp({
  enableHotkeys: true,
  initialTracks: [
    {
      metaData: { artist: "Artist", title: "Title" },
      url: "track.mp3",
      duration: 240
    },
    // ...
  ],
});
webamp.renderWhenReady(document.getElementById('app'));
```

### `nginx_refresh_deny`

**Назначение:** Автоматическое обновление списка заблокированных IP-адресов для nginx на основе access-логов.

**Описание:**
Скрипт анализирует все access-логи nginx (включая сжатые `.gz`) и создаёт файл `/etc/nginx/deny` со списком IP, которые:
- Ни разу не получали успешных ответов (200, 206, 304, 301)
- Не входят в разрешённые диапазоны (локальные сети: 192.168.x.x, 10.x.x.x, 172.16-31.x.x)

**Алгоритм:**
1. Собирает все IP с успешными статусами в `/tmp/nginx_allow`
2. Собирает все IP с неуспешными статусами в `/tmp/nginx_deny`
3. Удаляет из deny все IP, которые есть в allow или в локальных диапазонах
4. Формирует файл `/etc/nginx/deny` с директивами `deny IP;`
5. Проверяет конфигурацию nginx и перезагружает, если всё в порядке

**Использование:**
```bash
sudo ./nginx_refresh_deny
```

**Требования:** Права на запись в `/etc/nginx/` и возможность перезагрузки nginx.
